# ScienceClaw 重构 — 风险登记册

> **用途**: 持续跟踪项目风险，从"被动救火"转为"主动预防"
> **更新频率**: 每日站会更新，每周深度评审
> **触发条件**: 发现新风险、风险状态变化、风险发生

---

## 1. 风险状态定义

| 状态 | 含义 | 行动 |
|------|------|------|
| 🟢 已识别 | 风险已知，正在监控 | 定期 review |
| 🟡 预警中 | 概率或影响上升 | 制定缓解计划 |
| 🔴 已触发 | 风险已发生 | 启动应急预案 |
| ⚪ 已关闭 | 风险消除或接受 | 归档记录 |

---

## 2. 风险清单

### R1: SDK 依赖安装冲突

| 属性 | 内容 |
|------|------|
| **ID** | R1 |
| **来源** | P0-T3-9 |
| **描述** | `claude-agent-sdk` 与现有依赖（如 `langchain-core` 版本）可能存在冲突 |
| **概率** | 中（30%） |
| **影响** | 高 — 阻塞整个项目启动 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. 在隔离 Docker 环境预先测试安装<br>2. 准备 `requirements-claw.txt` 作为独立依赖清单<br>3. 保留 `requirements.txt.bak` 可回退 |
| **应急方案** | 若冲突无法解决，使用虚拟环境隔离 SDK |
| **负责人** | 后端负责人 |
| **下次 review** | Phase 0 结束 |

---

### R2: SSE 事件流与旧版不一致

| 属性 | 内容 |
|------|------|
| **ID** | R2 |
| **来源** | P3-T5-8 |
| **描述** | SDK `client.query()` 的消息格式与 LangGraph `astream()` 双模式差异可能导致 SSE 事件序列不一致 |
| **概率** | 高（60%） |
| **影响** | 高 — 前端崩溃或显示异常 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. 预留 3h 工时专门修复差异<br>2. 开发 SSE 对比测试脚本（P3-T5-9）<br>3. 前端预留兼容层（事件容错） |
| **应急方案** | 在 `ClaudeAgentEngine` 中增加事件适配层，强制转换格式 |
| **负责人** | 后端资深工程师 |
| **下次 review** | Phase 3 结束 |

---

### R3: OpenAI 兼容端点不支持

| 属性 | 内容 |
|------|------|
| **ID** | R3 |
| **来源** | P5-T4-3 |
| **描述** | Claude SDK 通过 `base_url` 对接 OpenAI 兼容模型时，可能因 API 差异导致功能异常 |
| **概率** | 中（40%） |
| **影响** | 中 — 部分模型（DeepSeek/Gemini）不可用 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. 混合架构保留 `DeepAgentEngine`<br>2. 非兼容模型自动回退到旧引擎<br>3. 用户界面提示"该模型暂由旧引擎支持" |
| **应急方案** | 完全回退到 `DeepAgentEngine` |
| **负责人** | 后端负责人 |
| **下次 review** | Phase 5 结束 |

---

### R4: 并发会话 Race Condition

| 属性 | 内容 |
|------|------|
| **ID** | R4 |
| **来源** | P6-T2-5, 痛点 #5 |
| **描述** | 子 Agent 全局字典（`GENERAL_PURPOSE_SUBAGENT`）在多会话并发时可能互相污染 |
| **概率** | 中（40%） |
| **影响** | 高 — A 用户的子 Agent 看到 B 用户的策略 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. P2-T3 实现会话级隔离的子 Agent<br>2. 使用 `asyncio.Lock` 保护共享状态<br>3. 并发测试覆盖（10+ 会话） |
| **应急方案** | 限制单实例并发会话数 |
| **负责人** | 后端资深工程师 |
| **下次 review** | Phase 2 结束 |

---

### R5: async bridge hack 移除导致崩溃

| 属性 | 内容 |
|------|------|
| **ID** | R5 |
| **来源** | P5-T9-2, 痛点 #4 |
| **描述** | 统一全异步架构后，现有同步调用点（如 `offload_middleware.py` 中的 `asyncio.run()`）可能崩溃 |
| **概率** | 高（70%） |
| **影响** | 高 — 工具调用链路断裂 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. P5-T9-1 先全面审计所有 sync→async bridge<br>2. 逐文件替换，每改一个文件立即测试<br>3. 保留必要的 shim 层（标注 `TODO: remove after full async`） |
| **应急方案** | 恢复 bridge 代码，标记为技术债务 |
| **负责人** | 后端中级工程师 |
| **下次 review** | Phase 5 结束 |

---

### R6: 灰度发布期间错误率超标

| 属性 | 内容 |
|------|------|
| **ID** | R6 |
| **来源** | P8-T4-5 |
| **描述** | 10% 灰度流量切换到新引擎后，生产环境错误率可能超过 1% |
| **概率** | 低（20%） |
| **影响** | 极高 — 用户流失、数据丢失 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. Staging 环境充分验证<br>2. 监控面板实时跟踪错误率<br>3. 准备一键回滚脚本（5 分钟内恢复） |
| **应急方案** | 立即执行 `./scripts/switch-agent.sh deep` 回滚 |
| **负责人** | DevOps |
| **下次 review** | Phase 8 开始 |

---

### R7: 团队学习成本导致进度延期

| 属性 | 内容 |
|------|------|
| **ID** | R7 |
| **来源** | 通用 |
| **描述** | 团队对 Claude SDK 不熟悉，实际开发效率低于预期 |
| **概率** | 中（35%） |
| **影响** | 中 — 进度延期 3-5 天 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. P0-T7-1 团队培训（1 天）<br>2. 先由资深工程师完成 P0-P2 核心，再知识传递<br>3. 预留 15% Buffer Time |
| **应急方案** | 延长开发周期或增加人手 |
| **负责人** | 项目经理 |
| **下次 review** | 每周 |

---

### R8: `deepagent/` 误修改导致后备不可用

| 属性 | 内容 |
|------|------|
| **ID** | R8 |
| **来源** | ADR-005 |
| **描述** | 开发者在不知情的情况下修改 `deepagent/` 文件，导致后备引擎损坏 |
| **概率** | 低（15%） |
| **影响** | 极高 — 失去回滚能力 |
| **状态** | 🟢 已识别 |
| **缓解措施** | 1. `constraints.md` 明确列为只读禁区<br>2. CI 检查 PR 中是否包含 `deepagent/` 文件变更<br>3. 代码审查强制检查 |
| **应急方案** | 从 git 历史恢复 `deepagent/` |
| **负责人** | 代码审查者 |
| **下次 review** | 每次 PR |
