# ScienceClaw 重构 — 每日站会模板

> **用途**: 标准化每日进度汇报，15 分钟内完成
> **参会人**: 执行者、审查者、项目经理（可选）
> **输出**: 更新后的 `progress.md` + 风险登记册（如有变化）

---

## 站会三问

每人回答以下三个问题（每问 2 分钟）：

### 1. 昨天完成了什么？

```
任务 ID: P?-T?-?
任务名称: 
完成标准: ✅ / ⏸️ / ❌
实际工时: Xh（预估 Yh，偏差 Zh）
```

### 2. 今天计划做什么？

```
任务 ID: P?-T?-?
任务名称: 
预计工时: Xh
依赖条件: （如等待审查、等待他人完成）
```

### 3. 有什么阻塞？

```
阻塞任务: P?-T?-?
阻塞原因: 技术 / 资源 / 决策 / 外部依赖
已尝试: 
需要帮助: 
风险升级: 否 / 是（如 2 小时内无法解决，升级至 R?）
```

## 站会检查清单

### 会前（每人 2 分钟准备）

- [ ] 查看 `progress.md` 确认自己负责的任务状态
- [ ] 更新实际工时（如果昨天有任务完成）
- [ ] 标记今天计划开始的任务为 🔄
- [ ] 检查是否有任务需要标记为 ❌（阻塞）

### 会中（15 分钟）

- [ ] 轮流回答三问（每人 5 分钟）
- [ ] 确认跨任务依赖（如 A 的输出是 B 的输入）
- [ ] 识别新风险 → 记录在 `risk-register.md`
- [ ] 确认里程碑是否按预期推进

### 会后（5 分钟）

- [ ] 更新 `progress.md`（所有人可见的最新状态）
- [ ] 更新 `risk-register.md`（如有新风险或状态变化）
- [ ] 发送站会纪要（阻塞项 + 今日重点）

## 站会升级规则

以下情况 **不得在站会中讨论**，需升级至专项会议：

| 情况 | 升级至 |
|------|--------|
| 技术方案争议（2 种以上实现方式） | 技术评审会 |
| 任务工时偏差 > 50% | 项目经理 + 技术负责人 |
| 新发现风险影响 > 中 | 风险评审会 |
| 里程碑可能无法按时达成 | 项目复盘会 |
| 需要修改 ADR | 架构决策会 |

---

## 特殊情况处理

### 异步站会（远程/跨时区）

- 在 `progress-tracking/` 目录下创建每日日志文件：`standup-YYYY-MM-DD.md`
- 每人按模板填写，截止时间当天 10:00 AM
- 项目经理汇总后发送至团队频道

### 周末/节假日

- 节前最后一次站会：确认节假日期间无阻塞任务
- 节后第一次站会：回顾节假日期间的任何变更

---

> **原则**: 站会不是解决问题的地方，是识别问题的地方。所有技术讨论会后进行。
